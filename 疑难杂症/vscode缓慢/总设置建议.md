## 配置说明

### 为什么 `build` 的处理不同？

| 配置项 | `build` 处理 | 原因 |
|--------|-------------|------|
| `files.exclude` | 不全局排除 `"**/build"` | 可能误伤名为 `build` 的源码目录 |
| `files.watcherExclude` | 仅排除特定项目的 build | 减少误伤 |
| `search.exclude` | 排除 `"**/build"` | 构建输出通常不需要搜索 |

**脑图项目的 build 建议**：
- 当前主项目 `pig/build` 只有 3MB，可保留在侧边栏可见
- 历史版本的 build 已在 `watcherExclude` 中排除

### 关于 `技术/脑图_keydog_贱狗_已经独立/log/`

- **2,968 个日志文件**
- **建议**: 仅 `watcherExclude`，不 `exclude`
  - 原因：你可能需要查看日志文件
  - 但不需要 VS Code 实时监视它们的变化

### 关于废弃项目

```json
"files.watcherExclude": {
  "技术/语言和他的编辑器和编译器teapug/**/废弃/**": true
}
```

- 不影响废弃项目在侧边栏的显示
- 但 VS Code 不会监视其中的文件变化
- 如果想完全隐藏，取消注释 `files.exclude` 中的对应行
