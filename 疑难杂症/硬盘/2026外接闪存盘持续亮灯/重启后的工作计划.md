# 重启后的工作计划

## 背景
- 海康 2TB SSD 已从 HFS+ 转换为 APFS
- 转换后磁盘访问极慢，命令超时
- 怀疑 Spotlight 索引死循环或权限问题
- 需要重启后诊断和处理

---

## 重启后立即执行（按顺序）

### 步骤 1: 检查磁盘是否自动挂载
```bash
diskutil list | grep -E "海康|disk7|disk4"
mount | grep "海康"
```

**预期结果**:
- 应该显示为 APFS 格式
- 应该已挂载到 /Volumes/2T海康CG视频

---

### 步骤 2: 测试访问速度
```bash
ls /Volumes/2T海康CG视频/
time ls /Volumes/2T海康CG视频/ | wc -l
```

**判断**:
- 如果秒级响应 → 正常
- 如果超过 10 秒或无响应 → 还有问题

---

### 步骤 3: 检查 Spotlight 状态（关键）
```bash
mdutil -s /Volumes/2T海康CG视频/
ps aux | grep -E "mds|spotlight" | grep -v grep
```

**如果显示 "Indexing enabled" 或 mds CPU 占用高**:
→ 立即执行步骤 4 关闭 Spotlight

---

### 步骤 4: 关闭 Spotlight 索引（如果卡死）
```bash
sudo mdutil -i off /Volumes/2T海康CG视频/
sudo mdutil -E /Volumes/2T海康CG视频/
sudo pkill -9 mds mds_stores
```

**然后重新测试访问速度**

---

### 步骤 5: 修复权限问题（根目录需要 sudo）

**检查当前权限**:
```bash
ls -ld /Volumes/2T海康CG视频/
ls -la /Volumes/2T海康CG视频/ | head -10
```

**如果根目录是 root:wheel，改为用户可写**:
```bash
# 方案 A: 修改挂载选项
sudo diskutil unmountDisk /dev/disk7
sudo diskutil mount -mountPoint /Volumes/2T海康CG视频 disk4s1

# 方案 B: 直接改权限（如果方案 A 不行）
sudo chown bergman:staff /Volumes/2T海康CG视频/
sudo chmod 755 /Volumes/2T海康CG视频/
```

**验证**:
```bash
touch /Volumes/2T海康CG视频/test_file
rm /Volumes/2T海康CG视频/test_file
echo "权限正常"
```

---

### 步骤 6: 深度检查（如果还是慢）

**检查是否有损坏的文件**:
```bash
# 查找可能损坏的文件
find /Volumes/2T海康CG视频/ -type f 2>&1 | grep -i "error" | head -20

# 检查磁盘健康
diskutil verifyDisk /dev/disk7
```

**如果 verifyDisk 报错**:
```bash
sudo diskutil repairDisk /dev/disk7
```

---

## 如果以上都失败（最后手段）

### 备选方案: 数据备份 + 抹盘

**步骤 1: 只读挂载备份数据**:
```bash
sudo mkdir -p /tmp/backup
sudo mount -o rdonly /dev/disk7s2 /tmp/backup  # 或 disk4s1

# 复制重要数据到电脑（分批，利用 162G 空间）
cp -r /tmp/backup/重要文件夹 /Users/bergman/Desktop/
```

**步骤 2: 抹盘重新格式化**:
```bash
sudo diskutil unmountDisk /dev/disk7
sudo diskutil eraseDisk APFS "2T海康CG视频" /dev/disk7

# 或者先分区再格式化
sudo diskutil partitionDisk /dev/disk7 GPT APFS "2T海康CG视频" 100%
```
